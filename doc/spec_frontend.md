# 前端介面設計

##　環境說明

- 手機優先，其次平板，可不必太考慮電腦
- 一頁式，除非使用者自行重整網頁，否則都只是 API 的更新
    - api 也並非主動即時更新，只有當使用者點選【更新資料】才會重新 get
    - 否則都會先把資料抓下來後，在不同地點和停車場靜態切換
- html、css、js分離
    - 其中 js 至少需要 config.js、main.js 和 map.js
- 可引入 tailwind、axis 套件
- 地圖採用 leaflet，可切換兩種底圖：台灣通用電子地圖(彩色)、衛星影像

## 前端介面說明

由上而下的區塊分別為

1. 問候區
2. 資料更新區
3. 常用地點
4. 停車場資訊
5. 地圖

## 1. 問候區

- 會有一個簡單的小圖片和一句問候的話
- 小圖片和問候語句會隨著時間變化，對應台灣時間
    - 00:00~06:00 凌晨 (midnight)
    - 06:00~11:00 早上 (morning)
    - 11:00~13:00 中午 (noon)
    - 13:00~18:00 下午 (afternoon)
    - 18:00~23:59 晚上 (night)
- 讓我可以彈性替換圖片和文字：
    - 文字設定 config.greetingText 物件，讓我可以替換 midnight ~ night 內的文字
    - 圖片存放：img 資料夾內，讓我自己存放 greeting_midnight ~ greeting_night 的 webp 圖片

## 2. 資料更新區

- 更新時間：顯示網頁最近一次 get api 的時間
- 更新資料：當使用者點擊此按鈕，才會重新 get 一次

## 3. 常用地點、4. 停車場資訊、5. 地圖

- 總共只有四個：首都羅東、公正國小、後站、前站
- 每個地點存放固定的停車場，也都有預設展示的停車場，請參考 `spec_api.md` 的對應清單
- 使用者切換常用地點時：
    - 預設展示的停車場資訊會改變
    - 地圖的底圖位置移動到該地點指定好的中心
    - 地圖上停車場的location icon會改變，變為顯示該地點的停車場
- 使用者切換站名的往左、往右的按鈕時：
    - 可以切換該地點下的停車場，即時資訊會跟著改變
    - 地圖的底圖位置不會移動(因為是在相同地點下切換)
    - 地圖上停車場的location icon會高量顯示使用者當前切換到的停車場
- 地圖上停車場的 location icon 時：
    - icon 會高亮
    - 上方的停車場名稱和即時資訊，會切換為使用者點擊的目標地點 (\*\*需討論邏輯合理性)
    - 以後可能會擴充屬性視窗，但目前先不放
